Intro to operator overloading

  • Thinking about the operation 1 + 2, it is essentially equivalent to add(1, 2).
  • For two user-defined ComplexNumber objects a and b , we would like to write a + b instead of a.add(b) or add(a, b).

✎  Mathematically speaking, an operator is just a function that receives one or more inputs and return one or more outputs.

✎  In C++, operators are implemented as functions.

      Note The phrase “In C++” is not redundant. We cannot overload operators in C, Assembly and Java.

  • The expression x + y is translated as the function call operator+(x, y).
  • Function overloading is used to determine that the compiler should call the double version of this function instead of the integer version.

Operators as functions

➥  Consider the following hypothetical snippet

#include <iostream>

class ComplexNumber
{
    double real;
    double imag;
public:
    ComplexNumber(double real, double imag) : real {real}, imag {imag} {}
};

int main()
{
    ComplexNumber a {42, 24};
    ComplexNumber b {24, 42};
    /*
    std::cout << a << '\n';     // these won't work.
    std::cout << b << '\n';
    std::cout << a + b << '\n';
    */
}
  • The compiler does not have a built-in version of the plus operator that it can use ComplexNumber operands. Thus, it will give an error.
  • We would need to write an overloaded function to tell the compiler how the + operator should work with operands of type ComplexNumber.

Operators as functions

➥  We can fix the code above:

#include <iostream>

class ComplexNumber
{
    double real;
    double imag;
public:
    ComplexNumber(double real, double imag) : real {real}, imag {imag} {}

    ComplexNumber operator+(const ComplexNumber& other) const {
        return ComplexNumber(real + other.real, imag + other.imag);
    }

    friend std::ostream& operator<<(std::ostream& out, const ComplexNumber& a) {
        out << a.real << " + " << a.imag << "*i";
        return out;
    }
};

int main()
{
    ComplexNumber a {42, 24};
    ComplexNumber b {24, 42};
    std::cout << a << '\n';     // these work now
    std::cout << b << '\n';
    std::cout << a + b << '\n';
    return 0;
}

Best practices in overloading operators

➜  Skip this slide. Students read later!

Best practices

  • An overloaded operator should operate on at least one program-defined type (either as a parameter of the function, or the implicit object).
  • When overloading operators, it’s best to keep the function of the operators as close to the original intent of the operators as possible.
  • If the meaning of an overloaded operator is not clear and intuitive, use a named function instead.
  • Operators that do not modify their operands (e.g. arithmetic operators) should generally return results by value.
  • Operators that modify their leftmost operand (e.g. pre-increment, any of the assignment operators) should generally return the leftmost operand by reference.

Overloading operators using normal functions

  • In the previous lesson, we overloaded operator+ as a friend function because we need access to internal members of the classes.

    friend Pennies operator+(const Pennies& p1, Pennies& p2) {
    // here we access "pennies" from p1 and from p2.
    return Pennies { p1.pennies + p2.pennies }; 
}

  • If we don’t need that access, we can write our overloaded operators as normal functions.

    Pennies operator+(const Pennies& p1, const Pennies& p2) {
    // get internal data via access function (getter function)
    return Pennies { p1.get_pennies() + p2.get_pennies() };
}

That’s the whole idea 🤨

Overloading operators using normal functions

  • Because normal and friend functions work almost identically (except different levels of access to private members), we generally won’t differentiate them.
  • The difference between friend function and normal function for overloading operators:
    • The friend function declaration inside the class serves as a prototype.
    • With the normal function version, we have to provide our own function prototype.

Overloading operator <<: Key points

✎  Overloading operator<< is similar to overloading operator+ (they are both binary operators), except that the parameter types are different.

✎  Consider the expression

std::cout << point;
  • the operator is <<
  • the left operand is the std::cout object – std::cout is an object of type std::ostream.
  • the right operand is the Point class object.

✎  Our overloaded function will look like this:

friend std::ostream& operator<< (std::ostream& out, const Point& point);

Overloading operator<<: Example

  • This is pretty straightforward. The notable difference is that std::cout has become parameter out (which will be reference to std::cout when the function is called).

Overloading operator<<: Key points

  • The notable difference is that std::cout has become parameter out (which will be reference to std::cout when the function is called).

  • The trickiest part here is the return type. If we try to return std::ostream by value, we will get a compiler error.

  • This happens because std::ostream specifically disallows being copied.

  • We return the left hand parameter as a reference:

    std::ostream& operator<< (std::ostream& out, const Point& point)

    • this not only prevents a copy of std::ostream from being made,

    • this also allows us to “chain” output commands together – recall lecture on return *this

Overloading operators using member functions

When overloading an operator using a member function:

  • The overloaded operator must be added as a member function of the left operand.
  • The left operand becomes the implicit *this object.
  • All other operands become function parameters.

Overload an operator as a friend or a member: Which should we use?

➥  Skip these slides – Students learn at home.

  • Not everything can be overloaded as a friend function

    The assignment (=), subscript ([]), function call (()), and member selection (->) operators must be overloaded as member functions, because the language requires them to be.

  • Not everything can be overloaded as a member function

    We are not able to overload operator<< as a member function. Because the overloaded operator must be added as a member of the left operand.

When to use a normal, friend, or member function overload

➥  Skip these slides – Students learn at home.

Rules of thumb

  • If you are overloading assignment (=), subscript ([]), function call (()), or member selection (->), do so as a member function.
  • If you are overloading a unary operator, do so as a member function.
  • If you are overloading a binary operator that does not modify its left operand (e.g. operator+), do so as a normal function (preferred) or friend function.
  • If you are overloading a binary operator that modifies its left operand, but you cannot add members to the class definition of the left operand (e.g. operator<<, which has a left operand of type ostream), do so as a normal function (preferred) or friend function.
  • If you are overloading a binary operator that modifies its left operand (e.g. operator+=), and you can modify the definition of the left operand, do so as a member function.
