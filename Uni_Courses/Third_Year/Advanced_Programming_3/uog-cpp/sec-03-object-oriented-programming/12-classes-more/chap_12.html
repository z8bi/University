The hidden this pointer

In Python, the first parameter of function member, normally named self, refer to the “current” object “under consideration”.

class Fraction:
    def __init__(self, nominator, denominator):
        self.nominator = nominator
        self.denomintory = denominator

Now, we learn the hidden pointer this which acts like self, but it is a pointer.

The hidden this pointer

✎  The keyword this is a const pointer that holds the address of the current implicit object.

✎  We use -> to select a member from a pointer to an object.

How is this set?

When we make the function call

simple.set_value(42);

the compiler rewrites the expression simple.set_value(42) as

// This is hypothetical explanation. 
// This line of code won't run in the main function.
Simple::set_value(&simple, 42);

this always pointers to the object being operated on

✎  Just like self variable in Python, each member function has a single this pointer parameter that points to the implicit object.

int main()
{
    Simple a{42};       // this = &a inside the Simple constructor
    Simple b{42};       // this = &b inside the Simple constructor
    a.set_value(12);    // this = &a inside member function set_value()
    b.set_value(21);    // this = &b inside member function set_value()

    return 0;
}

Explicitly referencing this

  • Most of the time, we don’t need to explicitly reference the this pointer.

  • If we have a parameter with the same name as a data member, we can disambiguate them by using this.

    struct Something
{
    double data{}; // not using m_ prefix because this is a struct

    void set_data(int data)
    {
        this->data = data;  // this->data is the member, 
                            // data is the local parameter
    }
};

Returning *this: function chaining

✎  Why we want to return *this – object pointed by this

  • It can sometimes be useful to have a member function return the implicit object as a return value.
  • This allows member functions to be chained together.
  • This is called function chaining (or method chaining).

✎  Common example we have used all the time:

  • Consider:

    std::cout << "Hello " << user_name;

  • The compiler evaluates the above snippet like this:

    (std::cout << "Hello, ") << user_name;

  • The expression (std::cout << "Hello, ") returns std::cout.

  • Then, std::cout is used again in std::cout << user_name.

Destructors to rescue

✎  A class Destructor is called automatically when an object of a non-aggregate class type is destroyed.

✎  Destructors are designed to allow a class to do any necessary clean up before an object of the class is destroyed.

✎  A class can only have a single destructor.

Destructor naming

  • The destructor must have the same name as the class, preceded by a tilde (~).
  • The destructor cannot take arguments.
  • The destructor has no return type.

An implicit destructor

Again, not a surprising:

  • f a non-aggregate class type object has no user-declared destructor, the compiler will generate a destructor, called implicit destructor, with an empty body.

Class template: Rationale

➥  Look at this scenario:

  • We define a Pair:

    struct Pair
{
    int first {};
    int second {};
}

  • Soon we recognize that we need a Pair with double data and possibly std::vector<double> data.

    struct Pair
{
    std::vector<double> first {};
    std::vector<double> second {};
}

➥  That’s enough to see the problem we encountered just like how we did with functions.

➥  We need a template to define class type (struct and class) so that we can instantiate different classes depending on their use cases.

Class template

A class template is a template definition for instantiating class types.

Class template

Just like with function templates, we

  • start a class template definition with a template parameter declaration
  • begin with the template keyword
  • specify all of the template types inside angled brackets <>
  • use the keyword typename for
  • continue to define class type (struct and class) as usual.

Friend non-member functions

  • We’ve used std::cout << some_object; so many times without noticing that std::cout is an object, << is an operator.
  • It works like object1 + object2 in which
    • object1 is std::cout
    • object2 is some_object
    • << plays the role of operator +.
  • If we write std::cout << polynomial, we wish std::cout can access private data members of polynomial to print meaning outputs to the consol.

➥  Here we go: we need the magic of friend.

Friendship is magic

  • Inside the body of a class, a friend declaration (using the friend keyword) can be used to tell the compiler that some other class or function is now a friend.
  • A friend is a class or function (member of other classes or non-member) that has been granted full access to private and protected members of another class

Key insight

Friendship is always granted by the class whose members will be accessed (not by the class or function desiring access).

Friend non-member functions

A friend function is a function (member or non-member) that can access the private or protected members of a class as though it were a member of that class.

Defining a friend non-member inside a class

✎  friend non-member functions can also be defined inside a class.

➥  print() is not a member of Accumulator. Since print() is defined as a friend, it is instead treated as a non-member function (as if it had been defined outside Accumulator).

Friend of multiple classes

✎  A function can be a friend of more than one class at the same time.

Prefer non-friend functions to friend functions

✎  We should prefer non-friend functions over friend functions.

➥  Look at this example: If we change the variable name value, we must change the body of print()

#include <iostream>

class Accumulator
{
private:
    int value { 0 }; // if we rename this data member

public:
    void add(int value) { this->value += value; } // we need to modify this

    friend void print(const Accumulator& accumulator);
};

void print(const Accumulator& accumulator)
{
    std::cout << accumulator.value; // and we need to modify this
}

int main()
{
    Accumulator acc{};
    acc.add(5); // add 5 to the accumulator

    print(acc); // call the print() non-member function

    return 0;
}

Prefer non-friend functions to friend functions

➥  A better idea to fix this issue is to add getter function value().

#include <iostream>

class Accumulator
{
private:
    int value { 0 };

public:
    void add(int value) { this->value += value; }
    int value() const { return value; } // added this reasonable access function
};

void print(const Accumulator& accumulator) // no longer a friend of Accumulator
{
    std::cout << accumulator.value(); // use access function instead of direct access
}

int main()
{
    Accumulator acc{};
    acc.add(5); // add 5 to the accumulator

    print(acc); // call the print() non-member function

    return 0;
}

Friend classes and friend member functions

A [friend class] is a class that can access the private and protected members of another class.

That’s it, just like that.

Friend classes

✎  A friend class is a class that can access the private and protected members of another class.

Friend classes: More information

  1. Friendship is not reciprocal

    Just because Display is a friend of Storage does not mean Storage is also a friend of Display – However, both can declare the other as friend.

  2. Class friendship is not transitive

    If A is a friend of B and B is a friend C, that does not imply A is a friend of C.

  3. Friendship is not inherited

    If A makes B a friend, classes derived from (child of) B are not friends of A.

Friend member functions

✎  Instead of making an entire class a friend, we can make a single member function a friend.

As this concept needs quite a lengthy explanation to make sense of the written code, this part is reserved for self-studied purpose.

Friend member functions: Example – Part 1

➥  Let us start with this simple code: Filename=friend_member_function_01.cpp.

class Storage

Friend member functions: Example – Part 2

➥  We exchange the def positions of Display and Storage: friend_member_function_02.cpp.

#include <iostream>

class Display
{
private:
    bool display_int_first {};

public:
    Display(bool displayIntFirst)
        : display_int_first { displayIntFirst }
    {
    }

    void display_storage(const Storage& storage) // compile error: compiler doesn't know what a Storage is
    {
        if (display_int_first)
            std::cout << storage.int_value << ' ' << storage.double_value << '\n';
        else // display double first
            std::cout << storage.double_value << ' ' << storage.int_value << '\n';
    }

class Storage
{
private:
    int int_value {};
    double double_value {};
public:
    Storage(int int_value, double double_value)
        : int_value { int_value }, double_value { double_value }
    {
    }

    // Make the Display::display_storage member function a friend of the Storage class

➥  We now have another problem: Because Display::display_storage() uses Storage as a reference parameter, and we just moved its definition below the definition of Display, the compiler will complain it does not know what a Storage is.

Friend member functions: Example - Part 3

➥  We fix all issues by the following steps:

  • add class Storage as a forward declaration so the compiler will be OK with a reference to `Storage
  • move the definition of Display::display_storage() out of the class, after the full definition of Storage class

Friend member functions: Example - Part 3

➥  Fully working code: friend_member_function_03.cpp

#include <iostream>

class Storage; // forward declaration for class Storage

class Display
{
private:
    bool display_int_first {};

public:
    Display(bool display_int_first)
        : display_int_first { display_int_first }
    {
    }

    void display_storage(const Storage& storage); // forward declaration for Storage needed for reference here
};

class Storage // full definition of Storage class
{
private:
    int int_value {};
    double double_value {};
public:
    Storage(int int_value, double double_value)
        : int_value { int_value }, double_value { double_value }
    {
    }

    // Make the Display::display_storage member function a friend of the Storage class
    // Requires seeing the full definition of class Display (as display_storage is a member)
    friend void Display::display_storage(const Storage& storage);
};

// Now we can define Display::display_storage
// Requires seeing the full definition of class Storage (as we access Storage members)
void Display::display_storage(const Storage& storage)
{
    if (display_int_first)
        std::cout << storage.int_value << ' ' << storage.double_value << '\n';
    else // display double first
        std::cout << storage.double_value << ' ' << storage.int_value << '\n';
}

int main()
{
    Storage storage { 5, 6.7 };
    Display display { false };
    display.display_storage(storage);

    return 0;
}
